## 📖 Repository Introduction

**Code-Vault-Python** is a personal, ever-growing collection of Python projects built to document my journey from *learning syntax* to *building real, working software*. Instead of leaving concepts in notebooks and half-finished tutorials, every idea here gets turned into a complete, runnable mini-project.

This isn't a dump of code snippets — it's a structured vault, updated as I learn, build, break things, and fix them again.

<br/>

## 🧭 About This Repository

This repository is where I apply what I learn in my **BCA (Data Science Specialization)** coursework and self-study to real, hands-on Python projects. Each project lives in its own folder, has its own documentation, and is built with the same care I'd put into a production tool — proper structure, error handling, and clean code.

<br/>

## 💡 Why I Built This Repository

> "You don't really know a concept until you've built something with it."

Reading about loops, file handling, or APIs is one thing — actually building a tool that uses them to solve a real (even small) problem is what makes it stick. This repository exists to:

- Turn passive learning into active, provable skill
- Build a portfolio recruiters can actually look through
- Track my own growth as a developer over time
- Force myself to think about structure, edge cases, and clean code — not just "does it run once"

<br/>

## 🎯 Learning Philosophy

<div align="center">

| Principle | What It Means Here |
|---|---|
| **Build > Memorize** | Every concept gets applied in a working project |
| **Small, Complete Projects** | Each project is finished, documented, and runnable — not half-done |
| **Consistency Over Intensity** | Regular small additions instead of rare big ones |
| **Readable Code First** | Code should be understandable before it's clever |
| **Document Everything** | If it's not documented, it's not really finished |

</div>

```bash
# Navigate into the repository
cd Code-Vault-Python
```

<br/>

## ▶️ Running Projects

Each project is self-contained inside its own folder under `Mini-Projects/`.

```bash
# Navigate to any project folder
cd Mini-Projects/Duplicate-File-Finder-CLI

# Check for a requirements.txt and install if present
pip install -r requirements.txt

# Run the project
python duplicate_file_finder.py
```

> Refer to each project's individual `README.md` for specific setup and usage instructions.
